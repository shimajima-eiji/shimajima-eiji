# 扉絵 v2 文字入れ依頼 (for 文記事)

ChatGPT (DALL-E / image_gen) 向け、note 記事の扉絵に文字入れ (コピー) を施す v2 制作依頼書。

## このフォルダの中身

- `README.md` — この依頼書 (ChatGPT に最初に読ませる)
- `v1.png` — 文字入れ前のベース画像 (Codex 生成済・1280×670)
- `article.md` — 対象記事の本文 (note 用版・読者像と主張を理解するため必読)

## 最初に実行してほしいこと

1. `v1.png` を画像として読み込む
2. `article.md` をテキストとして読み込む
3. 記事の主題・読者像・冒頭シーン・末尾の対話を理解する
4. v2 の文字入れ案を提案する

## v2 の役割

v1 (画像のみ) は完了している。v2 はそのベース画像にコピー (画像内テキスト) を入れて、note タイムラインで読者の指を止めさせる完成形を作る段階。

## あなた (ChatGPT) に判断してほしいこと

1. **コピーの内容**: 記事を読んで、note タイムラインで読者がスクロールを止め、クリックする理由になる 1 フレーズを考える。タイトル文字列の書き写しではなく、記事の主題から逆算したフレーズ
2. **コピーの字数**: 短い方が指は止まるが、何文字が適切かはあなたが判断
3. **コピーの配置**: v1 の構図を活かす配置 (上部の濃紺背景に余白あり) をあなたが選ぶ
4. **フォント・色・サイズ**: ベース画像と調和するか、敢えて対比させるかもあなたが判断

こちらからは構図・モチーフ・色・字数・配置の指示を一切しない。記事と v1 画像と読者像を渡すので、判断は委ねる。

## 出力仕様

- サイズ: 1280×670 (note 推奨)
- フォーマット: PNG
- ファイル名: `v2.png` (このフォルダに保存)

## 読者像 (判断材料)

- 読者はプログラミング学習中の人 (PHP / JavaScript 初学者〜中級者)
- 「for 文、書けるけど、ちゃんと説明できる気がしない」というモヤモヤを抱えている
- 自分のコードが「丸暗記で書いた」のか「仕様を理解して書いた」のかの判別がついていない
- 記事冒頭は **for 文と読者の対話劇** で始まる:
  - 読者「for 文さん、3 番目のところには何を書けばいいですか?」
  - for 文「いや、3 番目に *何を書くか* は、決まってないんですよ」
- v1 画像はその「3 スロット + 任意の文」を抽象立体で見せている
- v2 のコピーは、その鏡を見た読者に「これは自分の話だ」と最後の一押しをする役割

## 過去版

- v1 (画像のみ・コピーなし): `v1.png` (subagent 評価: 主題視覚化 good / 禁止モチーフ混入なし / v2 文字入れ余白あり)

## 追加禁止 (記事に書かれていないもののみ)

- v1 と同じ禁止 (合格・脳・歯車・電球・笑顔人物・コード画面・拳・鉢巻き)
- 記事タイトル『for文【for(これ;これ;これ)】の内容を正しく説明できますか?』を画像内テキストでそのまま書かない
  - 理由: タイトルはスクロール時に既に表示されている。サムネで重複させると情報が薄まる
- 「初心者必見」「簡単」「マスター」「3 分でわかる」など煽り文句
  - 理由: 記事は仕様の核心を扱っており、煽りで信頼が落ちる

## 成功基準

- 読者が note タイムラインをスクロールしている最中、サムネで指が止まる
- 止まった瞬間に「for 文の 3 番目に何書くか、自分はちゃんと説明できるか?」と内心問われる
- コピーを 3 秒読んだだけで記事を開きたくなる
- 記事を開いた読者が、本文末尾「A・B・C は任意の文」を読んだとき、サムネのコピーと記事の核心が同じことを指していると気づける
- 過去版 (v1) と差別化されている (= コピーが入っただけでなく、コピーによって絵全体の意味が変わっている)

## 完成後の差し戻し方法

`v2.png` をこのフォルダ (またはあなたが指定する任意の場所) に保存して、依頼者 (nomuraya) に「v2 完成」と伝える。依頼者側で:

1. CDN (GitHub Releases) に upload
2. amazonphotos にバックアップ
3. note 記事の frontmatter と本文の URL を v1 → v2 に差し替え
4. note.com に再 publish (update)

を実行する。
